# Session Recap — 2026-05-11

## What was built

### Step 1 — Next.js scaffold
- App scaffolded with TypeScript, Tailwind, App Router, `src/` dir
- `vault/` was temporarily moved aside to allow `create-next-app` to run (it rejects non-empty dirs)

### Step 2 — Supabase schema
- Supabase CLI installed as local dev dep (`npx supabase`)
- Migration at `supabase/migrations/20260511083143_initial_schema.sql`
- Tables: `tag_categories`, `tags`, `transactions`, `transaction_tags`, `push_subscriptions`, `app_state`
- RLS enabled on all tables, allow-all policies (single-user app)
- `app_state` seeded with `last_synced_at = 1970-01-01` and will store `powens_user_token` after connect

### Step 3 — Bank connector
- `src/lib/bank/connector.ts` — `BankConnector` interface
- `src/lib/bank/gocardless.ts` — GoCardless implementation (kept but unused)
- `src/lib/bank/powens.ts` — **active** Powens implementation

### Step 4 — Sync + Webhooks
- `src/app/api/sync/route.ts` — manual "Sync now" (POST), pulls last 90 days from Powens
- `src/app/api/webhook/powens/route.ts` — **main entry point**: Powens calls this after each auto-sync (event: `ACCOUNT_SYNCED`), verifies Bearer token, pulls new transactions, sends push notifications
- Netlify scheduled function **removed** — Powens owns the schedule

### Step 5–6 — Transactions UI + Manual entry
- `src/app/api/transactions/route.ts` — GET (with date range) + POST (manual entry)
- `src/app/api/transactions/[id]/tags/route.ts` — PUT to set tag per category
- `src/app/api/transactions/[id]/associate/route.ts` — POST to copy tags from manual → bank tx and delete manual entry
- `src/app/api/tag-categories/route.ts` — GET categories with nested tags
- `src/components/TransactionRow.tsx` — per-transaction row with tag pickers + manual match suggestion
- `src/components/TagPicker.tsx` — per-category tag selector
- `src/components/ManualEntryForm.tsx` — toggle form to enter a spending manually

### Step 7 — Charts
- `src/components/SpendingCharts.tsx` — one Recharts pie chart per tag category (expenses only)

### Step 8 — PWA
- `public/manifest.json`, `public/sw.js`
- `src/app/api/push/route.ts` — stores push subscriptions
- `src/components/ServiceWorkerRegistrar.tsx` — registers SW + requests notification permission

### Step 9 — Connect flow (Powens-specific)
- `src/app/connect/page.tsx` — one-time bank linking UI
- `src/app/api/connect/route.ts` — creates Powens permanent user, generates webview temp code, returns webview URL
- After bank connection, page displays `POWENS_USER_TOKEN` for user to copy to `.env`

## Bank connector switch: GoCardless → Powens
- Powens auto-syncs on its own schedule and pushes `ACCOUNT_SYNCED` webhook → no cron needed
- Webhook handler at `/api/webhook/powens` is the trigger; it then pulls via Powens REST API
- Auth: simple Bearer token check (acceptable risk for single-user personal app)
