# Session Recap — 2026-05-12

## What was done

### Fix: Powens sync 500 error

- Root cause 1: Powens `/users/me/transactions` requires a mandatory `limit` parameter — missing it returns `{"code": "missingParameter", ...}`
- Root cause 2: `coming=false` was sent as a string; Powens expects `coming=0`
- Fix: switched to paginated fetching (500 per page, offset loop) and changed `coming` to `0`
- Added response body to error messages for easier future debugging

**File**: `src/lib/bank/powens.ts`

### Feature: Password authentication

Single-password protection for the whole app. Stack:
- `jose` (npm) for JWT signing — HS256, 7-day expiry
- `src/lib/session.ts` — `encrypt()` / `decrypt()` helpers
- `src/proxy.ts` — Next.js Proxy (replaces deprecated `middleware.ts`) guards all routes
- `src/app/api/auth/login/route.ts` — POST, checks `APP_PASSWORD` with `crypto.timingSafeEqual`, sets `budget_session` httpOnly cookie
- `src/app/api/auth/logout/route.ts` — POST, clears cookie
- `src/app/login/page.tsx` — minimal password form
- `src/app/page.tsx` — "Log out" button added to header

Public routes (bypass auth):
- `/login`
- `/api/auth/login`
- `/api/auth/logout`
- `/api/webhook/powens` (has its own Bearer token check)

**New env vars required**:
```
APP_PASSWORD=<your password>
SESSION_SECRET=<run: openssl rand -base64 32>
```

Both must be set in Netlify dashboard env vars for production.
