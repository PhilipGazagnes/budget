### Fix: duplicate transactions in dev DB

Root cause: the `/connect` flow had been run **3 times** on the sandbox, creating 3 Powens connections (ids 3, 4, 6) all pointing to the same bank account (same IBAN). Each connection produced its own set of transaction records with distinct Powens IDs, so the `external_id` upsert dedup couldn't catch them.

Fix:
- Deleted connections 3 and 4 via `DELETE /2.0/users/me/connections/{id}`
- Truncated `transactions` and `transaction_tags` on dev DB
- Re-synced — 218 transactions, 0 duplicates

### Fix: Incremental sync using last_synced_at

**Problem**: "Sync now" always fetched the last 90 days from Powens on every run, regardless of when the previous sync happened.

**Fix**: The sync route now reads `last_synced_at` from `app_state` before computing `fromDate`:
- If set → use that date as `fromDate` (only fetch what's new since last sync)
- If not set (first-ever sync) → fall back to `-90 days`

`last_synced_at` was already being written at the end of each sync, so no schema changes were needed.

**File**: `src/app/api/sync/route.ts`
